# 노무관리 Pro v5 — 백업 운영 매뉴얼

> 백업 담당자에게 전달하는 운영 절차서.
> 이 매뉴얼대로만 진행하면 데이터 사고에 대비할 수 있습니다.

---

## 0. 개요 — 왜 백업을 받아야 하나

노무관리 Pro v5는 **모든 데이터를 Supabase(외부 클라우드 DB)에 저장**합니다. Supabase가 정상이면 별도 백업 없이도 운영 가능하지만, 다음 시나리오는 백업이 없으면 복구 불가합니다:

- Supabase 자체 장애 (데이터센터 사고, 디스크 손상, 회사 폐업 등)
- 실수로 테이블 삭제 / 데이터 wipe
- 7일 이상 지난 시점의 데이터 복구 (Supabase Pro 자동 백업은 7일치만 보관)

따라서 **주 1회 별도 백업**을 받아 외부에 보관하는 것이 안전합니다.

## 2. 백업 받는 방법 (단계별)

### 2-1. 관리자 페이지 접속
1. 브라우저에서 https://noprohr.netlify.app 접속
2. 관리자 이메일·비밀번호로 로그인
3. 관리자 페이지로 자동 이동됨 (검은색 다크 테마)

### 2-2. 백업 페이지로 이동
1. 좌측 사이드바에서 **[💾 백업/복구]** 메뉴 클릭
2. "📅 백업 상태" 카드에서 마지막 백업 시점 확인
   - "오늘 백업됨 ✓" 초록색: 정상
   - "마지막 백업: N일 전" 노란색: 곧 백업 필요
   - "백업 기록 없음" 또는 "7일 이상 전" 빨간색: **즉시 백업 필요**

### 2-3. 백업 실행 (둘 중 택1)

#### 방법 A: 전체 회사 일괄 백업 (권장)
1. 우측 상단 **[⬇ 전체 회사 일괄 백업]** 버튼 클릭
2. 확인 창에 "예" 선택
3. 회사 수만큼 JSON 파일이 자동 다운로드됨 (브라우저 다운로드 폴더)
4. 완료 후 알림창에 "성공 N개 / 실패 0개" 표시 확인
5. **실패가 있으면**: 해당 회사를 개별 백업으로 재시도

#### 방법 B: 회사별 개별 백업 (특정 회사만)
1. 페이지 하단 회사 목록에서 해당 회사 행의 **[📥 백업]** 버튼 클릭
2. 즉시 JSON 파일 다운로드

### 2-4. 다운로드 확인
- 파일명 형식: `nopro-backup-{회사명}-{YYYYMMDD}.json`
- 예: `nopro-backup-홍길동상사-20260430.json`
- 파일 크기: 보통 100KB ~ 수 MB (회사 규모에 따라)

## 3. 백업 파일 보관 가이드

### ⚠️ 보안 주의 — 필수 준수
백업 파일에는 다음 민감 정보가 포함됩니다:
- 직원 이름, 연락처, 주민번호 앞자리 (평문)
- 주민번호 뒷자리 (암호화 상태)
- 모든 출퇴근 기록, 급여 내역
- 회사 사업자 정보

**개인정보보호법 적용 대상**입니다. 유출 시 과징금·형사 책임 가능.

### ✅ 권장 보관 방법

| 보관처 | 안전도 | 비고 |
|-------|-------|------|
| **암호화된 외장 USB/HDD** | ⭐⭐⭐⭐⭐ | 가장 안전. BitLocker, VeraCrypt 등 사용 |
| **암호화된 클라우드 폴더** | ⭐⭐⭐⭐ | Google Drive 비밀번호 보호 폴더, OneDrive Personal Vault |
| **잠긴 PC의 별도 폴더** | ⭐⭐⭐ | 윈도우 BitLocker 활성화 PC만 |
| ❌ **이메일 첨부** | ❌ | **절대 금지** (수신자 PC 유출 위험) |
| ❌ **공용 PC** | ❌ | **절대 금지** |
| ❌ **카카오톡/메신저 전송** | ❌ | **절대 금지** |

### 보관 기간
- **최소 4주분 보관** (사고 발견까지 시간 차 고려)
- 4주 초과한 옛 백업은 삭제 권장 (정보 보존 의무 < 유출 위험)
- 단, 정기 점검·감사 등 보관 의무가 있다면 그에 따름

### 폴더 구조 예시
```
백업/
├── 2026-04월/
│   ├── 2026-04-29 (월요일 백업)/
│   │   ├── nopro-backup-회사A-20260429.json
│   │   ├── nopro-backup-회사B-20260429.json
│   │   └── ...
│   ├── 2026-04-22/
│   └── 2026-04-15/
├── 2026-05월/
└── ...
```

## 4. 백업 누락 시 알림

관리자 페이지에 자동 알림이 표시됩니다:

| 위치 | 표시 |
|------|------|
| 사이드바 [백업/복구] 메뉴 | 빨간 점 + "!" 또는 "Nd" (N일 경과) |
| 대시보드 상단 | 노랑/빨강 경고 배너 (7일 초과 시) |
| [백업/복구] 페이지 | "마지막 백업: N일 전" 빨간 글씨 |

알림이 보이면 **즉시 백업 받기**.

## 5. 사고 발생 시 복구 절차

### 5-1. 사고 인지 시 가장 먼저 할 일
1. **추가 변경 즉시 중단** — 사용자에게 시스템 사용 중지 요청
2. **사고 시점 기록** — 언제, 어떤 화면에서, 어떤 작업 후 발생했는지
3. **개발자(서비스 운영사)에 즉시 연락** — 본 매뉴얼만으로 자체 복구 시도하지 말 것

### 5-2. 복구 시나리오별 대응 (참고용)

| 시나리오 | 복구 방법 | 담당 |
|---------|---------|------|
| 한 직원의 출퇴근 기록만 잘못됨 | 사용자가 직접 재입력 | 사용자 |
| 한 회사의 일부 데이터 손실 (1~6일 전) | Supabase Pro PITR 또는 audit_log 복원 | **개발자** |
| 한 회사의 일부 데이터 손실 (7일 초과) | 백업 파일 + audit_log 조합 | **개발자** |
| Supabase 자체 장애 | 백업 파일에서 전체 복원 | **개발자** |

**모든 데이터 복원은 개발자가 SQL 콘솔에서 진행합니다.** 백업 담당자는 백업 파일을 안전하게 전달만 하면 됩니다.

### 5-3. 백업 파일 전달 시 주의
- 이메일 첨부 금지
- 메신저 전송 금지
- **암호화된 압축파일 + 별도 채널로 비밀번호 전달** (예: zip 비밀번호는 전화로)
- 전달 후 자기 PC에서도 임시 폴더 정리

## 6. 자주 묻는 질문

### Q1. 백업 받는 데 시간이 오래 걸려요
- 회사 수가 많으면 30초~1분 정도 걸립니다 (회사당 0.5초 + 다운로드 시간)
- 화면을 닫거나 다른 페이지로 이동하지 말고 끝날 때까지 기다리세요

### Q2. 백업 도중 실패 메시지가 떠요
- 보통 네트워크 일시 불안정 또는 서버 일시 부하
- 1~2분 후 재시도하면 대개 해결
- 계속 실패하면 개발자 연락

### Q3. 백업 파일을 열어봐도 되나요?
- 텍스트 에디터로 열림 (JSON 형식)
- **단, 절대 수정 후 저장하지 마세요** — 복구 시 파일 무결성 깨짐
- 내용 확인이 필요하면 복사본을 만들어서 확인

### Q4. 매주 백업하기 번거로운데 자동화 안 되나요?
- 가능합니다. 다만 추가 개발이 필요하므로 개발자와 협의 후 결정
- 자동화 도입 시에도 한 달에 1회 정도는 수동 점검 권장

### Q5. 백업을 안 받은 지 한 달이 넘었어요
- Supabase Pro PITR 7일치 + 그 이전은 복구 불가 영역으로 들어감
- **즉시 백업** 후, 그 사이 사고 없었는지 audit_log 점검
- 향후 정기 알림 설정(예: 매주 월요일 9시 알람) 권장
